ПОРЯДОК ЗАПУСКА BAT-ФАЙЛОВ ДЛЯ ПЕРЕВОДА

ВАЖНО:
- Перед первым запуском '2_extract_strings.bat' сделайте резервную копию папки '../data_to_py'!
- Перед первым запуском '3_create_translation_map.bat' сделайте резервную копию папки '../data_to_py'!
- Убедитесь, что Python установлен и добавлен в переменную среды PATH.
- Установите необходимые библиотеки: pip install colorama astunparse lxml customtkinter pygments deep_translator

ШАГИ:

1.  `1_disassemble_dat.bat`
    -   Запустите этот файл первым.
    -   Он запросит путь к папке с оригинальными .dat файлами игры.
    -   Дизассемблирует .dat файлы в .py формат и поместит их в папку '../data_to_py'.
    -   Создаст лог '../LogDisassembler.txt' с информацией об ошибках (если были).
    -   Если возникнут ошибки 'KeyError: (X, Y)', добавьте соответствующие команды в '../disasm/ED9InstructionsSet.py' и запустите этот bat-файл снова, пока все файлы не будут успешно дизассемблированы.

2.  `2_extract_strings.bat`
    -   Запустите этот файл ПОСЛЕ успешного выполнения шага 1.
    -   Он проанализирует .py файлы в папке '../data_to_py'.
    -   Извлечет строки, подходящие для перевода, в файл '../data_game_strings.xliff'.
    -   Создаст начальную карту строк '../strings_map.json'.
    -   !!! ВАЖНО: Этот скрипт НЕ изменяет .py файлы в '../data_to_py'. !!!

3.  **ПЕРЕВОД ФАЙЛА XLIFF**
    -   Откройте файл '../data_game_strings.xliff' в редакторе XLIFF (OmegaT, Poedit и т.д.) или текстовом редакторе.
    -   Введите переводы в теги `<target>`.
    -   Сохраните файл.

4.  `3_create_translation_map.bat`
    -   Запустите этот файл ПОСЛЕ перевода и сохранения '../data_game_strings.xliff'.
    -   Он прочитает переведенный .xliff файл.
    -   Создаст/обновит файл '../strings_map.json', содержащий соответствие "оригинальная_строка" -> "переведенная_строка". Этот JSON будет использоваться на следующем шаге.

5.  `4_compile_dat.bat`
    -   Запустите этот файл ПОСЛЕ создания/обновления '../strings_map.json'.
    -   Он запросит, компилировать ли ВСЕ файлы или ТОЛЬКО те, для которых есть перевод.
    -   Для каждого выбранного .py файла:
        -   Создаст временный .py файл с подставленными переводами из '../strings_map.json'.
        -   Скомпилирует временный .py в .dat файл.
        -   Переместит готовый .dat файл в папку '../py_to_data'.
        -   Удалит временный .py файл.
    -   По завершении в папке '../py_to_data' будут находиться скомпилированные .dat файлы с переводом.
